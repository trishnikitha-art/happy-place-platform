# CONNECTOR_POLICY — Constitutional Specification (P1)

**Role:** Answers one question — **who owns bytes?** Every connector implements this contract.
**Status:** Normative governance. Enforceable. Supports deterministic replay.
**Evidence tags:** [E:file]=committed audit.

---

## 1. Purpose
Govern every integration that touches provider-owned data (Google Drive, Gmail, Calendar, Forms, Stripe, Maps). Establish that providers own source bytes/records; HPOS holds references and HPOS-generated derived artifacts only.

## 2. Connector Contract
Every connector implements exactly:
```
Acquire → Canonicalize → EvidenceEnvelope → Emit Event
```
Nothing else. No business logic, no mutation of provider truth. [E:governance/CONSTITUTIONAL_GOVERNANCE_AUDIT.md Observation primitive]

## 3. Copy — FORBIDDEN
Downloading provider-owned source bytes into HPOS-owned storage as a **copy** is prohibited. Drive file bytes, Gmail messages, Calendar event bodies remain in the provider.

## 4. Reference — DEFAULT
Store only: `provider`, `resourceId`, `url`, `contentType`, `metadata`. HPOS references; it does not hold the original. [E:primitives/CONSTITUTIONAL_PRIMITIVE_CATALOG.md Reference]

## 5. Hybrid — ALLOWED (Derived only)
HPOS **may** create derived artifacts of a referenced original (e.g., WEBP/AVIF variants, thumbnails of a Drive image). These are:
- generated by HPOS (Derived Authority),
- content-hashed,
- linked back to the source `reference` via `sourceReference`.
This is not a copy of the original — it is a transformation HPOS owns and can regenerate.

## 6. Provider Authority
Owns: source bytes, source records, source truth. HPOS observes; it does not override.

## 7. HPOS Authority
Owns: canonical objects, derived artifacts (variants), operational schedule, relationships.

## 8. Derived Authority
HPOS-generated transformations of referenced bytes. Identified by `contentHash` + `sourceReference`. Regenerable from the original reference; never the source of truth. [E:primitives/DUPLICATE_PRIMITIVE_REPORT.md D5 Evidence]

## 9. Replay Requirements
- Connector emissions are `EvidenceEnvelope`s; replay rebuilds **references + derived artifacts** from evidence.
- Replay NEVER re-copies source bytes; it re-links references and regenerates derived artifacts from `sourceReference`.

## 10. Required Metadata (per emission)
`connectorId`, `provider`, `resourceId`, `contentType`, `contentHash` (derived only), `sourceReference` (derived only), `observedAt`, `witness`.

## 11. EvidenceEnvelope Requirements
- Must carry `provider` + `resourceId`.
- For derived artifacts: `sourceReference` + `contentHash` (lineage).
- Conforms to EVENT_SCHEMA_POLICY canonical envelope.

## 12. Certification
A connector is **certified** when:
1. default behavior is Reference (not Copy);
2. Hybrid only for Derived artifacts with `contentHash` + `sourceReference`;
3. every emission is an `EvidenceEnvelope` satisfying EVENT_SCHEMA_POLICY;
4. replay reconstructs references + derived artifacts without re-copying source.
→ Resolves "who owns bytes": Provider owns bytes; HPOS owns references + derived transformations.
